# Docker Troubleshooting Interview Questions & Answers

## Introduction

Docker troubleshooting is one of the most frequently tested areas during DevOps interviews. Interviewers are not only interested in Docker commands but also in your ability to identify production issues, analyze root causes, and resolve incidents efficiently.

This document contains commonly asked Docker troubleshooting interview questions with practical answers based on real-world production scenarios.

---

# 1. What is the first command you execute when a Docker issue is reported?

### Answer

The first command I execute is:

```bash
docker ps
```

This helps determine:

- Whether the container is running
- Container uptime
- Port mappings
- Current status

If the container is not running, I immediately execute:

```bash
docker ps -a
```

to identify stopped or crashed containers.

---

# 2. How do you troubleshoot a container that keeps restarting?

### Answer

My troubleshooting approach is:

1. Check container status

```bash
docker ps -a
```

2. Review logs

```bash
docker logs <container>
```

3. Inspect container configuration

```bash
docker inspect <container>
```

4. Verify environment variables

5. Validate application configuration

6. Rebuild and redeploy if necessary

The root cause is usually:

- Application crash
- Incorrect CMD
- Missing environment variables
- Dependency failure
- Database unavailable

---

# 3. What is the difference between docker logs and docker inspect?

### docker logs

Shows application output generated by the running process.

Example:

```bash
docker logs nginx
```

Useful for:

- Application exceptions
- Startup failures
- Runtime errors

---

### docker inspect

Displays Docker metadata.

Includes:

- Network settings
- Environment variables
- Mounted volumes
- Restart policy
- Health status
- Container IP

Example:

```bash
docker inspect nginx
```

---

# 4. A container starts successfully and exits immediately. Why?

### Answer

Docker containers continue running only while the primary process is active.

Common reasons include:

- Incorrect CMD
- Wrong ENTRYPOINT
- Shell script finishes execution
- Foreground process missing

Correct Example:

```dockerfile
CMD ["nginx","-g","daemon off;"]
```

---

# 5. How do you troubleshoot port conflicts?

### Answer

Check running containers.

```bash
docker ps
```

Verify listening ports.

```bash
ss -tulpn
```

or

```bash
netstat -tulpn
```

Stop the conflicting service or map a different host port.

Example:

```bash
docker run -p 8080:80 nginx
```

---

# 6. How do you troubleshoot networking issues?

### Answer

Commands used:

```bash
docker network ls
```

```bash
docker network inspect bridge
```

```bash
docker exec nginx ping google.com
```

```bash
docker exec nginx nslookup google.com
```

Check:

- Network connectivity
- DNS resolution
- Gateway configuration
- Container IP

---

# 7. How do you investigate storage problems?

### Answer

Commands:

```bash
docker volume ls
```

```bash
docker volume inspect myvolume
```

Verify:

- Mount paths
- Named volumes
- Bind mounts
- File permissions

---

# 8. What causes "Image Pull Access Denied"?

### Answer

Possible causes:

- Wrong image name
- Private repository
- Authentication failure
- Missing permissions

Resolution:

```bash
docker login
```

Verify repository name and user permissions.

---

# 9. How do you troubleshoot high CPU or Memory usage?

### Answer

Use:

```bash
docker stats
```

Also inspect application logs and identify resource-intensive processes.

Possible solutions:

- Increase resource limits
- Optimize application
- Scale horizontally
- Fix memory leaks

---

# 10. Explain Docker HEALTHCHECK.

### Answer

HEALTHCHECK periodically verifies whether the application inside the container is functioning correctly.

Example:

```dockerfile
HEALTHCHECK CMD curl --fail http://localhost || exit 1
```

Benefits:

- Detects unhealthy containers
- Improves availability
- Helps orchestration platforms restart failed containers

---

```bash

```

Check:

- Docker DNS
- Network configuration
- Host DNS

# 12. What is docker exec used for?

### Answer

Allows administrators to execute commands inside a running container.

Example:

```bash
docker exec -it nginx /bin/sh
```

Useful for:

- Debugging
- Configuration validation
- Log inspection
- Connectivity testing

---

# 13. How do you clean Docker disk usage safely?

### Answer

Commands:

```bash
docker system df
```

Review usage first.

Then execute:

```bash
docker system prune
```

Avoid deleting production volumes unless backups are available.

# 16. What is your troubleshooting workflow during a production incident?

### Answer

I follow a structured workflow:

1. Understand the reported issue.
2. Verify container status.
3. Review logs.
4. Inspect container metadata.
5. Validate networking.
6. Verify volumes.
7. Check CPU and Memory usage.
8. Identify the root cause.
9. Apply the fix.
10. Validate application functionality.
11. Monitor the container after recovery.

This structured approach minimizes downtime and avoids unnecessary changes.

---

# 17. How do you debug a container without restarting it?

### Answer

Use:

```bash
docker exec -it <container> /bin/sh
```

Then:

- Verify files
- Check environment variables
- Test connectivity
- Review running processes
- Validate configuration

---

# 18. Why should you always check logs before restarting a container?

### Answer

Restarting a container may erase valuable debugging information.

Logs often contain:

- Exception stack traces
- Startup failures
- Missing dependencies
- Configuration errors

Always collect evidence before restarting.

---

# 19. What Docker command do you use most frequently during incidents?

### Answer

The most commonly used commands are:

- docker ps
- docker logs
- docker inspect
- docker exec
- docker stats
- docker network inspect
- docker volume inspect

These commands cover the majority of production troubleshooting scenarios.

---

# 20. What advice would you give to someone learning Docker troubleshooting?

### Answer

- Learn Linux fundamentals first.
- Understand Docker architecture.
- Read logs before taking action.
- Follow a structured troubleshooting process.
- Practice debugging real-world issues.
- Build hands-on labs with intentionally broken containers.
- Focus on identifying the root cause instead of applying temporary fixes.

---

# Interview Tips

✔ Explain your troubleshooting workflow instead of listing commands.

✔ Mention root cause analysis during answers.

✔ Provide production examples whenever possible.

✔ Demonstrate familiarity with Docker logs, inspect, exec, networking, and storage.

✔ Show that you prioritize evidence collection before making changes.

# Final Takeaway

Docker troubleshooting is not about memorizing commands. It is about following a logical process to identify, analyze, and resolve issues efficiently. Strong troubleshooting skills are one of the most valuable qualities of a successful DevOps Engineer and are frequently assessed in technical interviews.
