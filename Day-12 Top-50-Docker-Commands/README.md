# Day 12: Top 50 Docker Commands

Part of the **100 Days of DevOps** series.

## What's Covered

A practical, production-oriented reference of the 50 Docker commands DevOps engineers use daily — organized by category, with real-world context (not just syntax dumps).

## File Guide

| File | Contents |
|---|---|
| `docker-cli-cheatsheet.md` | Quick-reference table of all 50 commands |
| `image-commands.md` | Building, tagging, pulling, pushing images |
| `container-commands.md` | Running, managing, inspecting containers |
| `volume-commands.md` | Volume creation, mounting, backup/restore |
| `network-commands.md` | Docker networking commands and modes |
| `compose-commands.md` | Docker Compose command reference |
| `registry-commands.md` | Login, tag, push to ECR / Docker Hub |
| `system-commands.md` | `docker system`, `docker info`, disk usage |
| `cleanup-commands.md` | Pruning images, containers, volumes, networks |
| `production-tips.md` | Do's and don'ts for using these commands in prod |
| `interview-questions.md` | Scenario-based Docker command interview Q&A |
| `command-comparison.md` | Commonly confused commands, side by side |
| `real-time-scenarios.md` | On-call/debugging situations mapped to commands |
| `sample-outputs.md` | Real terminal output examples for key commands |

## How to Use This

1. Skim `docker-cli-cheatsheet.md` first for the full list.
2. Go deep into the category file relevant to your task.
3. Use `real-time-scenarios.md` when debugging live issues.
4. Use `interview-questions.md` before interviews.
